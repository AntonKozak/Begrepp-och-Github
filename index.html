Github flow

Github flow är en av bästa verktyg som man har gjort. Ide av github flow är att man kan skaffa ett projekt i github och alla andra som jobbar med projektet kan kopiera original (main branch) projekt till sin dator och skaffa ny branch som gör att man kan jobba med kopia av main(original) proektet på sin lokala dator eller skaffa ny branch på filwebben och skriva kod där.
1. Ladda ner main projekt till sin dator.
2. Skaffa ny branch (local kopia) med namn och börja jobba med projekt.
3. Efter man gjorde ändring i ny branch så man ska göra git add –all. Den här funktionen ska skicka din färdiga kod som är klart och ska inte ändras mer till staging enviroment.
4. När du gjorde git add så man kan kolla git status som vissa vad är finns att skicka till repositori i github.
5. Efter följer commit kommand i commit man skriver kort förklaring vad ändrades i projektet.
6. Push kommand skicka alla ändringarna till repositori.
7. I repositori man ska valja sitt branch och göra merg kommand som kopplar main branch med ny branch som man skaffade.
8. Alla andra som jobbar samtidigt med proektet kan göra pull request som ladda nya ändringar till lokal dator (för att undvika konflikts).
Det var snabbt och väldigt enkel förklaring hur funkar github flow. En viktigt grej är att man ska undvika att jobba samtidigt på en fil, annars man för merge issue eller pull request issue.

Flexbox

Flexbox ger lättare väg att manipulera html elements. Man kan sätta dem på olika nivå, storlek eller centralt på sida, jämna allt eller göra vad man vill. Ide är att göra bästa placering av elements på webbsidan beroende på skärm storleken eller fönster storleken för att använda max plats. Man skaffa div element med class namn och i css skriva man kommendera för att manipulera med div element.

DOM

DOM är objekt som innehåller reglar för struktur och innehåll till webbsida. Med hjälp av DOM vi kan ändra webbsida som vi vill och dator kan läsa och jobba med sida. Vi kan manipulera sida med hjälp av JavaScript. DOM som är träd har kopplingar till alla elementar, attributer, text eller annat objekt.

Fin kod

När vi skriver kod vi måste tänka om att skriva bra strukturerad kod som innebär lättläslig och bra förståelse. En annan person eller även den person som skrev koden måste ha lättförståelse för att när jobbet klart försvinner minne om av betyder den function eller variable. Viktigt att kommentera och sätta rätt namn. Bra planering och några år av erfarenhet kan hjälpa mycket. Bättre några dagar planera och skriva en gång rätt att istället skriva om koden några gånger och har mäss till slutet. Varje företag har sina egna reglar om kodstruktur viktig att hålla regler.

Ramverk

Ramverk och kodbibliotek är verktyg som hjälper att skriva kod snabbare och enklare. Finns många ramverk och kommer nya så måste man göra bedömning av vilken man ska använda. React, Angular, Vue är mest populärt och användande för idag. Ramverk eller kodbibliotek man ska lära sig som nytt programering språk.
